## Überblick

MinJ (Minimalistic Java) ist eine Skriptsprache, die ich entworfen und gebaut habe, um zu verstehen, wie Interpreter von innen funktionieren. Kein Spielzeug-REPL, das Arithmetik auswertet — MinJ hat Klassen, Methoden, typisierte Variablen, Schleifen, Listen und `return` mit ordentlichem Stack-Unwinding. Das Ganze läuft auf der JVM: eine 246-Zeilen-ANTLR4-Grammatik generiert Lexer und Parser, ein 730-Zeilen-`EvalVisitor` traversiert den Parse-Tree und führt ihn aus, und das Ergebnis wird als einzelnes fat-JAR ausgeliefert, das überall läuft, wo Java läuft.

Ich habe MinJ im dritten Semester gebaut. Das Ziel war, jede Schicht einer Sprachimplementierung zu verstehen — lexikalische Analyse, Parsing, Scoping, Typprüfung, Methodenaufruf — indem ich eine baue, statt nur darüber zu lesen.

## Wie es funktioniert

Quellcode durchläuft drei Stufen:

1. `MinJ.g4` definiert die Grammatik. Während des Gradle-Builds generiert ANTLR4 daraus `MinJLexer` und `MinJParser`. Der Parser erzeugt einen typisierten Parse-Tree.
2. `EvalVisitor` erweitert ANTLRs `MinJBaseVisitor<Object>` und überschreibt eine Methode pro Grammatikregel. Jeder `visit*`-Aufruf traversiert den Baum und führt aus: Variablenzuweisung, Arithmetik, Kontrollfluss, Methodenaufruf, Objektinstanziierung.
3. `Main.java` liest eine `.mj`-Datei, schickt sie durch Lexer und Parser und ruft `visitor.visit(tree)` auf.

Grammatik und Ausführungslogik sind strikt getrennt. Die `.g4`-Datei definiert, wie MinJ-Programme aussehen; `EvalVisitor` definiert, was sie tun. Ein neues Sprachkonstrukt hinzuzufügen heißt, genau zwei Dateien anzufassen.

Unter der Haube nutzt die Laufzeit `Cell`-Objekte, die für jede Variable den Wert, den deklarierten Typ, Veränderlichkeit und den Dynamisch-Flag tracken. `return`-Anweisungen werfen ein `ReturnSignal` (eine RuntimeException-Unterklasse), um den Call-Stack abzuwickeln — einfach, und damit funktionieren Returns korrekt aus verschachtelten Schleifen und Bedingungen heraus. Objektinstanzen kopieren die Feld-Cells ihrer Klasse bei der Konstruktion, sodass jede Instanz ihren eigenen Zustand hat.

## Die Sprache

MinJ hat `var` (veränderlich, dynamisch typisiert), `val` (unveränderlich, einmalige Zuweisung zur Laufzeit erzwungen) und explizite Typen (`int`, `float`, `double`, `boolean`, `char`, `String`). Typisierte Deklarationen werden bei der Zuweisung geprüft — wer `int x` deklariert und einen String zuweisen will, bekommt einen Fehler.

Kontrollfluss: `if`/`elseif`/`else`, `while`, `for i = 1 to n step s` und `foreach x in list`. Alle Blöcke werden mit `end` abgeschlossen. Boolesche Operatoren akzeptieren sowohl Symbole als auch Wörter (`&&` oder `and`, `||` oder `or`, `!` oder `not`).

Klassen haben Felder, Methoden und `new`-basierte Instanziierung mit Punkt-Aufruf-Syntax. Globale Funktionen verwenden `func` und unterstützen mehrere Rückgabewerte mit Destrukturierung (`var summe, produkt = addAndMultiply(3, 4)`). Kommentare funktionieren mit `//`, `#` oder `/* */`.

## Beispiele

### FizzBuzz

```minj
val limit = 100
for i = 1 to limit do:
  if i % 15 == 0 then:
    print "FizzBuzz"
  elseif i % 3 == 0 then:
    print "Fizz"
  elseif i % 5 == 0 then:
    print "Buzz"
  else:
    print i
  end
end
```

### Klassen und Methoden

```minj
class Counter:
  var count = 0
  method inc():
    count = count + 1
  end
  method get():
    return count
  end
end

var c = new Counter()
c.inc()
c.inc()
print c.get()   // 2
```

### Rekursive Fakultät

```minj
func factorial(n):
  if n <= 1 then:
    return 1
  end
  return n * factorial(n - 1)
end

print factorial(5)   // 120
```

## Tooling und CI/CD

Der Gradle-Build nutzt das ANTLR-Plugin, um Lexer und Parser bei jedem Build aus `MinJ.g4` neu zu generieren. ShadowJar packt alles (ANTLR-Runtime inklusive) in ein einzelnes fat-JAR — beliebige `.mj`-Dateien lassen sich mit `java -jar minjc-0.3.0.jar datei.mj` ausführen.

Eine GitHub-Actions-Pipeline läuft bei jedem Push auf den `release`-Branch: Code auschecken, JDK 21 aufsetzen, fat-JAR bauen, Tests ausführen, Beispiele bündeln und ein versioniertes Release-Asset veröffentlichen. Die Dokumentation wird bei jedem Push auf `master` automatisch auf GitHub Pages deployt.

## Wie man die Sprache erweitert

Ein neues Sprachkonstrukt hinzuzufügen ist ein mechanischer Vorgang. Am Beispiel der `while`-Schleife:

1. Grammatikregel in `MinJ.g4`:
```antlr
whileStmt : WHILE expr DO COLON block END ;
```

2. In der `statement`-Regel registrieren:
```antlr
statement : varDecl | assign | printStmt | ifStmt | whileStmt ;
```

3. Lexer-Token:
```antlr
WHILE : 'while' ;
DO    : 'do' ;
END   : 'end' ;
```

4. Visitor-Override in `EvalVisitor.java`:
```java
@Override
public Object visitWhileStmt(MinJParser.WhileStmtContext ctx) {
    while ((Boolean) visit(ctx.expr())) {
        visitBlock(ctx.block());
    }
    return null;
}
```

5. Neu bauen: `./gradlew clean shadowJar`
