## Übersicht

Ich betreibe einen Linux-VPS, auf dem sowohl meine alltäglichen Dienste als auch die Backends meiner Nebenprojekte laufen. Jeder Dienst ist containerisiert — pro Kategorie gibt es eine eigene Docker-Compose-Datei, sodass Updates, Rollbacks und Fehlersuche voneinander isoliert bleiben. Massenspeicher liegt auf Remote-Storageboxen statt auf der VPS-Festplatte, damit Rechen- und Speicherkosten getrennt bleiben.

## Routing und TLS

Traefik steht als Reverse-Proxy vor allen Diensten. SSL-Zertifikate werden automatisch über Let's Encrypt ausgestellt und erneuert, HSTS ist durchgängig aktiv, und HTTP wird auf HTTPS umgeleitet. Zertifikate verwalte ich nicht von Hand.

## Dienste

| Dienst | Zweck |
|---|---|
| **Nextcloud** | Dateisynchronisation und Cloud-Speicher, angebunden an eine Remote-Storagebox |
| **Navidrome** | Musik-Streaming — liest direkt aus der Nextcloud-Musikbibliothek |
| **Matrix / Synapse** | Verschlüsselter Messaging-Homeserver |
| **Element Web** | Web-Client für Matrix |
| **Coturn** | TURN-Server für Sprach- und Videoanrufe über Matrix |

## Entwicklungs-Deployments

Auf demselben VPS laufen auch Entwicklungs-Builds meiner Projekte:

- **Fortalis Auth** – Authentifizierungs-Microservice
- **Fortalis Backend** – Spiel-Backend-API
- **Schachserver** – Backend für meine Schach-Engine
- **Dieses Portfolio** – wird ebenfalls von diesem Host ausgeliefert

## Speicherarchitektur

Rechenleistung und Speicher sind bewusst getrennt. Der VPS übernimmt die Verarbeitung, Remote-Storageboxen halten die Daten (Nextcloud-Dateien, Medienbibliotheken). So lässt sich die Speicherkapazität unabhängig vom Server erweitern, und die Kosten bleiben kalkulierbar.
